TrustCheck verifies before agents delegate.

A zero-trust behavioral verification layer for agent delegation. TrustCheck turns a capability claim into real HTTP canary tests, signed evidence, and an independently verifiable receipt.

Live Health View Contract Public Key Proof Bundle

What it is

TrustCheck verifies whether a consented target can actually perform one declared capability before another agent delegates consequential work.

The invoice-total capability is the Hello World demonstration. The product is the verification pattern.

What it is not

  • Not identity verification.
  • Not AgentFacts schema validation.
  • Not a reputation system.
  • Not a universal trust score.
  • Not production-ready security infrastructure.

Verification flow

The same pattern can generalize to any capability that can be expressed as a contract, canaries, predicates, evidence, and a signed receipt.

Security boundary

Independent verification proves that the saved receipt was signed by the active TrustCheck key and that its signed fields were not modified.

It does not prove that a compromised issuer holding the private key could not forge receipts. TrustCheck is a validated hackathon prototype, not a production-ready certificate authority, transparency log, or universal trust system.
