What it is

TrustCheck acts as an active security gate. Before a delegator agent assigns a high-consequence task, TrustCheck runs runtime canary calls against the target agent's live endpoint, verifies predicates against a behavioral contract, and stamps a tamper-resistant cryptographic receipt.

The invoice-total capability is the Hello World demonstration. The product is the reusable zero-trust verification pattern.

What it is not

  • Not identity verification (doesn't assume endpoint identity means capability accuracy).
  • Not simple schema validation (executes real runtime canaries).
  • Not a reputation system or historical trust scoring.
  • Not production-ready certificate authority infrastructure.

Zero-Trust Verification Pipeline

How TrustCheck secures the delegation handoff between two autonomous agents.

01. Claim
Target Capability
Target agent claims it satisfies a contract.
02. Contract
Behavioral Spec
Machine-readable rules, endpoints, and input/output predicates.
03. HTTP Test
Runtime Canary
TrustCheck makes real, isolated sandbox calls to target.
04. Evidence
Payload Hashing
Canary responses are collected and hashed into evidence.
05. Receipt
Ed25519 Signing
TrustCheck signs the receipt with its private key.
06. Verify
Offline Check
Delegator validates receipt signature using the public key.
07. Decision
Delegation Action
Safe delegation executes if signature verifies and verdict is PASS.

Verify Locally in Terminal

You can verify the signature of any receipt file using our standalone, zero-dependency Python verification script.

Run the verification script directly from your terminal using the published well-known keys:

BASE="https://trustcheck-nandahack-293749289787.northamerica-northeast1.run.app"

# Verify Alpha (PASS)
python verify_receipt.py proof/alpha-pass/receipt.json --well-known "$BASE/.well-known/trustcheck-key.json"

# Verify Beta (FAIL)
python verify_receipt.py proof/beta-fail/receipt.json --well-known "$BASE/.well-known/trustcheck-key.json"

Security Boundary Notice

Independent cryptographic verification proves that the receipt was issued by the holder of the active TrustCheck private key, and that the verdict and hashes have not been altered. It does not prevent key compromise or issuer-level forge attempts. TrustCheck is a validated hackathon prototype demonstrating zero-trust capabilities validation, not a commercial certificate authority.
